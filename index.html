Irie Composer rewrite & about ARM and macOS support May 20, 2024

Turns out my code for irie composer's UI and functionalities was very, very attatched to windows, in a way that even trying to cross-compile it is incredibly painful.

Therefore I've began a boring process of recoding it entirely and building it under a linux distro in a VM and constantly cross-compiling to windows and building for the linux distro I'm in.

About ARM and macOS

I don't have any apple devices so I can't realiably test and develop Irie Composer for macOS. Maybe one day I'll get my hands on a cheap macbook and code it to support macOS? if that ever happens, you can be sure I'll code support for it to work under ARM hosts aswell. the thing is, only way to have OpenGL 3.3 on ARM is to emulate it which doesn't offers the most optimal performance

Sure, Irie Composer isn't a grafically intense application, but it wouldn't make sense to provide ARM builds when the program itself uses emulation to run on the platform, I wouldn't want that in my text editor.

That's it for news. it's a lot harder to support multiple operating systems and architectures than I had originally imagined.

A feather-like text editor March 7, 2024

Text editors nowadays (the fancy ones like vscode, sublime, and whatever is the new cool editor in the block) all have the same issue: they can't seem to use little resources, they're all amazingly heavy. So, I decided to ditch out electron and all the weird frameworks for a code editor written from scratch inspired by no other than Netscape Composer (at least.. the name Composer)

I shall present.. the new.. Irie Composer!

yeah.. not finished so its not that worth to already start comparing it to the modern ones.. but hey, its going there. UI was written from scratch with OpenGL, and text rendering being done using Multi Signed Distance Fields (MSDF) But not to worry, it sounds heavy but its really efficient, quite more than mesh based font rendering. Fonts used are the same as the websites, with the exception the editor uses MPLUS Code instead of IBM Plex Mono.

Anyway, that's enough for today. I'll keep posting updates on this project here and eventually give its own page once it becomes usable.
